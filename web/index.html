usdl2

Native SDL2 subset for Python — import usdl2. Pure C for MicroPython (unix + windows), CircuitPython unix, and CPython (python3 / python.exe / Android). Public names are SDL2 symbols only: functions, constants, and macros such as SDL_DEFINE_PIXELFORMAT.

Android APKs install native android_21_* wheels from TestPyPI via pydisplay_android. When this module is not linked or installed, pydisplay falls back to add_ons/usdl2.py.

Source on GitHub pydisplay_android cmods

Build

Needs libSDL2 at build time

MicroPython unix

Install devel headers, then link as a usermod sibling.

sudo apt install libsdl2-dev
cd micropython/ports/unix
make USER_C_MODULES=../../..

./micropython/ports/unix/build-standard/micropython \
  usdl2/test_usdl2.py

MicroPython windows

Point SDL2_DEV at the official MinGW devel ZIP (not vendored here).

export SDL2_DEV=../../other/SDL2-2.30.10
cd micropython/ports/windows
make USER_C_MODULES=../../..

# Or from cmods:
./build_mp.sh --port windows --variant standard

CPython

Editable install on unix; Windows needs the MSVC SDL2 tree + SDL2.dll on PATH.

pip install -e .
xvfb-run -a python3 test_usdl2.py

Android wheels

CI publishes android_21_arm64_v8a and android_21_x86_64 (cp313/cp314) to TestPyPI. APK packaging lives in pydisplay_android.

export ANDROID_HOME=~/.buildozer/android/platform/android-sdk
export ANDROID_NDK_HOME=~/.buildozer/android/platform/android-ndk-r28c
./scripts/ci_prepare_sdl2_android.sh
pipx run cibuildwheel --platform android
