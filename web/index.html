PyDevices / usdl2

Native SDL2 subset for pydisplay's desktop backend

usdl2 is a focused native binding to a useful subset of SDL2 for pydisplay's desktop backends. It gives MicroPython and CPython the same window, surface, and input primitives on unix and Windows without pulling in a full SDL wrapper surface or fighting pygame-only paths.

Use it when you want the pydisplay desktop experience — live windows, pointer and keyboard events — backed by C. Pair it with the cmods build matrix for MicroPython firmware tests, or install the CPython module where your host tooling already lives.

Build with libSDL2

# Linux / unix port
sudo apt install libsdl2-dev
git clone https://github.com/PyDevices/usdl2.git

# Windows (MinGW) -- point at the official SDL2 dev zip
export SDL2_DEV=~/SDL2-2.30.10

# CPython / Android: TestPyPI usdl2, where applicable
